# Simple Stack VM

Минималистичная стековая виртуальная машина на языке C. В текущей версии добавлена поддержка управления потоком исполнения (Jumps), что позволяет реализовывать полноценные циклы и условные переходы.

## Особенности
- **Стековая архитектура**: все операции выполняются над данными в стеке.
- **Таблица диспетчеризации**: использование массива указателей на функции (`actions[]`) для быстрого декодирования инструкций.
- **Управление потоком (Flow Control)**: поддержка команд ветвления `JMP` и `JMZ`.
- **Безопасность**: автоматическая проверка Stack Overflow и Stack Underflow перед выполнением каждой команды.

## Система команд

### Арифметика и Стек
- `BC_PUSH <val>` — положить 64-битное число в стек.
- `BC_POP` — удалить верхний элемент.
- `BC_DUP`, `BC_SWAP` — дублирование и обмен элементов стека.
- `BC_IADD`, `BC_ISUB`, `BC_IMUL`, `BC_IDIV` — стандартная арифметика.
- `BC_INEG` — смена знака числа.

### Ввод-вывод и Сравнение
- `BC_IPRINT` — вывести верхнее число из стека в `stdout`.
- `BC_IREAD` — считать число из `stdin` и положить в стек.
- `BC_ICMP` — сравнить два верхних числа (результат в стеке: -1, 0 или 1).

### Swap и duplicate
- `BC_SWAP` — поменять числа на стеке местами.
- `BC_DUP` — положить на стек копию числа.

### Управление потоком (Новое!)
- `BC_JMP <offset>` — безусловный переход на указанное смещение относительно текущего `IP`.
- `BC_JMZ <offset>` — переход по смещению, если на вершине стека 0 (извлекает значение из стека).
- `BC_STOP` — остановка программы.

## Пример программы (Цикл)
Программа считывает число и в цикле прибавляет к нему 2, пока результат не станет равен 42:
```c
const union ins program[] = {
    { BC_IREAD },
    { BC_DUP },
    { .as_arg64 = {BC_PUSH, .arg = 42} },
    { BC_ICMP },
    { .as_arg64 = {BC_JMZ, .arg = 2} }, // Если ввели 42 сразу — прыгаем к STOP
    { BC_STOP },
    { .as_arg64 = {BC_PUSH, .arg = 2} }, // Точка входа в цикл
    { BC_IADD },
    /* ... логика дублирования и проверки ... */
    { .as_arg64 = {BC_JMZ, .arg = -7} }, // Прыжок назад в цикл
    { BC_STOP }
};
```
